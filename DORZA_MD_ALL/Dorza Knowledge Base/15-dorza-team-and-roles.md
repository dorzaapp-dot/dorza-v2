# Dorza — Team & Roles

> Last updated: March 2026 | Two founders, bootstrapped, pre-launch.

## Founders

### Mahir — CEO / Product & Commercial Lead

**Background:** 10 years of product experience at large media organisations. Understands how to build and ship digital products at scale, and how to position them for specific audiences.

**Owns:**
- Client acquisition — canvassing, pitching, closing deals in Sydney suburbs
- Client relationships — onboarding meetings, monthly check-ins, upsells, retention
- Product vision and roadmap — what to build next, which features matter, pricing decisions
- Brand, marketing, and Dorza's own digital presence
- Content QA — final review of AI-generated websites, social posts, chatbot responses before they go live
- Partnerships — accountants, business chambers, referral networks
- Finance — invoicing, cash flow, pricing experiments

**Day-to-day (launch phase):** Out canvassing 3–4 days per week. Running client onboarding, reviewing website builds, approving social content. Refining the pitch based on what works. Managing the first 10–20 client relationships personally.

### Adi Widodo — CTO / Tech Lead

**Background:** Experienced backend engineer and tech lead. AI experience. Long-term engineer who wants to lead the technical side of the business.

**Owns:**
- Internal automation OS — orchestration (n8n / Make.com) connecting intake → build → deploy → schedule
- AI pipeline engineering — Claude API integrations, prompt management, content generation workflows
- Hosting and infrastructure — Vercel / AWS, Cloudflare, deployment scripts, CI/CD
- Chatbot platform — Claude API wrapper, knowledge base management, widget deployment
- Social media automation — Buffer API integration, scheduled content pipeline
- Google Business API — automated profile creation, review monitoring
- Client dashboard backend (future) — APIs for analytics, content approval, support tickets
- Security, monitoring, uptime — keeping all client sites running reliably
- Form handling backend — webhook/API for form submissions (currently email-based, to be formalised)

**Day-to-day (launch phase):** Building the automation infrastructure. Setting up the initial tooling. Deploying client websites generated by Claude Code. Monitoring site uptime. Iterating on the technical pipeline based on what breaks or slows down during the first 10 clients.

## The frontend/design gap

Neither founder naturally covers the client-facing visual output layer — what customers actually see and judge Dorza on. This includes website template quality, social media graphics, the client dashboard UI, and Dorza's own visual identity.

**Current approach:**
- Claude Code and Lovable fill the gap for the first 10–20 clients
- AI-generated output is "good enough" for local businesses where the bar is low
- Mahir does visual QA — developing an eye for what works and flagging what doesn't

**Plan to close the gap (at ~10 clients):**
- Contract a freelance designer/developer part-time
- Brief: 3–5 website templates per industry vertical, social media design system, client dashboard UI
- Budget: ~$2–4K upfront for template library + $500/mo retainer for ongoing design work

**Alternative:** Adi expands into frontend work alongside infrastructure. This is a conversation to have — many backend engineers can do competent frontend, it's just not their preference.

## Decision-making authority

| Domain | Final say | Mechanism |
|--------|-----------|-----------|
| Client-facing (pricing, features, messaging, sales process) | Mahir | Informs Adi, but Mahir decides |
| Technical (architecture, tools, deployment, security) | Adi | Informs Mahir, but Adi decides |
| Strategic (hiring, spending >$500, pivots, new markets) | Both | Must reach consensus |
| Prompt engineering (build prompts, content templates) | Both | Weekly 30-min review session |

## Shared responsibilities

- Strategic decisions — pricing changes, new segments, when to hire
- Prompt engineering — refining Claude Code build prompts, content generation templates, chatbot configs. This is Dorza's core IP. Schedule weekly 30-min sessions to review what's working and iterate.
- Process optimisation — reviewing the 7-phase pipeline, identifying bottlenecks, improving turnaround time

## Working together: first 10 clients

The first 10 clients should be deliberately messy. Both founders do everything, stepping on each other's toes, because both need to understand the full pipeline before cleanly separating.

- Adi should sit in on 2–3 client meetings to understand what clients care about
- Mahir should watch Adi deploy a site and set up automation to understand technical constraints
- Both should review every client website before it goes live
- Daily 15-min sync: what happened, what's blocked, what's next

## Working together: after 10 clients

The separation should be clean. The test: can each founder go a full workday without needing the other to unblock them?

- Adi should be able to take a completed intake.md and deliver a live site without asking Mahir anything
- Mahir should be able to run a client meeting, review a site, and approve content without touching code
- Communication shifts from daily syncs to async updates + weekly strategy session

## Hiring plan

| Trigger | Hire | Role |
|---------|------|------|
| 10 clients, revenue covers cost | Freelance designer | Templates, graphics, visual QA |
| 30+ clients, Mahir can't manage all relationships | Part-time account manager | Client check-ins, content approval, support |
| 50+ clients, Adi needs help with infrastructure | Junior developer | Site deployments, monitoring, routine maintenance |
| 100+ clients | Full-time operations person | End-to-end client delivery pipeline |

All hires are contingent on revenue justifying the cost. Bootstrapped means every hire must pay for itself within 2 months.
